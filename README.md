📈 목표가 Upside 랭킹 V3
네이버 목표가 + Kiwoom HTS 실시간 데이터 기반 종목 상승여력 분석 프로그램

이 프로젝트는 다음 두 가지 데이터를 결합하여
종목별 상승여력(Upside)을 계산하고 랭킹을 생성하는 GUI 프로그램입니다.

네이버 금융 목표주가

키움증권 HTS 실시간 데이터(opt10007 + opt10001)

최적화 버전에서는 네이버 목표가가 존재하는 종목만 HTS에서 조회하여
속도와 안정성을 크게 개선했습니다.

🚀 주요 기능
✔ 1. 네이버 목표가 멀티스레드 수집
KOSPI/KOSDAQ 전체 종목 대상

ThreadPoolExecutor 기반 병렬 크롤링

약 3,000개 종목도 빠르게 처리

✔ 2. 목표가가 있는 종목만 HTS에서 조회 (최적화 핵심)
기존: 전체 종목 3,000개 TR 요청

변경: 목표가 존재 종목만 TR 요청 (약 400~600개)

TR 요청량 80% 감소 → 속도 5~7배 향상

✔ 3. Kiwoom TR 안정화
opt10007 → opt10001 순차 처리

TR 간격 1200ms 적용

이벤트 누락/멈춤 현상 방지

✔ 4. 단위 표기 정확화
Kiwoom opt10007의 단위 특성을 반영:

항목	단위
거래대금	백만원 단위
상장주식수	천주 단위
시가총액	원 단위 → 억원으로 변환


GUI/CSV에 명확히 표기:

거래대금(백만원)

상장주식수(주)

시가총액(억원)

✔ 5. GUI 기반 실시간 진행률 표시
네이버 목표가 수집 진행률

HTS TR 진행률

목표가 존재 종목 수 표시

✔ 6. CSV 자동 저장
날짜 기반 파일명 생성

UTF-8 with BOM 저장

엑셀에서 바로 열림

🧠 프로그램 동작 흐름
코드
1. KOSPI/KOSDAQ 전체 종목 목록 수집
2. 네이버 목표가 멀티스레드 수집
3. 목표가가 있는 종목만 필터링
4. Kiwoom TR(opt10007 → opt10001) 순차 조회
5. 상승여력(Upside) 계산
6. GUI 테이블 출력 + CSV 저장
📊 상승여력(Upside) 계산 방식
코드
Upside(%) = (목표가 - 현재가) / 현재가 × 100
목표가가 없는 종목은 자동 제외됩니다.
